What is this?

This series of scripts will help with moving all of your mapped network drives (J: and so on) from your old computer to your new computer.

It will:

  1. Get all of the network drives from your old computer
  2. Put the drive letters and addresses into a text file
  3. Read this text file and map all of your drives on your new computer

How do I do it?

On your old computer

  1. Download this script and save it somewhere on your computer
  2. Right-click on (Get-DriveAddresses.ps1) and click on "Run with PowerShell" to run it
    If this doesn't work, talk to Dyota or Lucas
  3. It should have created a file called DriveAddresses.csv, in the same folder
  4. Take this file outside of your computer somehow (put it on a USB, email it to yourself, etc.)

On your new computer

  1. Download this other script and save it somewhere on your computer
  2. Put your file DriveAddresses.csv in the same folder
  3. Before doing the next step, double check that you have no drive letter naming conflicts.
    • If you need to change drive names, open up DriveAddresses.csv directly and change the drive letters in there
    • For example, if you already have a J: Drive on your new computer, make sure that your DriveAddresses.csv file doesn't have a J: on it - replace that J with another letter
  4. Right-click on the script file (Set-DriveAddresses.ps1) and click on "Run with PowerShell" to run it
    If this doesn't work, talk to Dyota or Lucas
  5. Check My Computer / This PC. You should have all of your previous drives mapped on your new computer.

What happens if "Run with PowerShell" doesn't work?

You will need to open PowerShell and run the scripts from there

If it still doesn't work, the Execution Policy will need to be changed. Check the docs here.

Can do a one-off unblock by running Unblock-File -Path ".\Get-DriveAddresses.ps1"
